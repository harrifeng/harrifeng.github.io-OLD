数据库概述

数据库理论基础

  • 数据库泛型就是数据库应该遵循的规则, 数据库泛型也叫做"范式", 常用的四种范式是:
    • 第一范式(1NF):属性不再可以分割
    • 第二范式(2NF):满足1NF的基础上, 不允许出现有field依赖于主键的一部分
    • 第三范式(3NF):在2NF的基础上,不允许出现"可传递的依赖关系"
    • BCN范式(BCNF):在3NF的基础上, 不允许主键的一部分被主键另一部分或其他部分决定\

数据库技术构成

  • SQL分为三个部分来定义:
    • DDL语句: 用来定义数据库,表,试图,索引,触发器等等.比如CREATE, ALTER, DROP
    • DML语句: 对某个表的操作, 比如INSERT, SELECT, UPDATE
    • DCL语句: 控制用户的访问权限:,比如GRANT, REVOKE等
  • 要使用数据库访问技术才能让SQL在数据库中起作用:
    • 微软使用了ADO.NET来访问数据库. 如果使用ADO.NET访问MySQL的话要安装驱动程序 Connector/Net
    • Java使用了JDBC(Java Data Base Connectivity)来访问数据库. 如果使用Java访问 MySQL的话要安装驱动程序Connector/J
  • Mysql的GA是Generally Available的意思,表示此版本可以应用到生产环境了.

Windows平台下安装与配置MySQL

Linux平台下安装与配置MySQL

  • Linux 打开,关闭,重启,状态查询的命令如下 
    # > /etc/init.d/mysql start|stop|restart|status
  • 安装完成后,需要拷贝一个/usr/share/mysql下的cnf文件,到/etc/文件夹下,改名叫做my.cnf
  • 在my.cnf里面常规的有这么几个设置要点:
    • [client][mysql]是客户端的设置 [mysqld]是服务器端的设置
    • password = "passwd", 设置这个password那么就不需要每次输入密码了
    • port=3306, 设置数据库的端口号,默认是3306
    • default-storage-engine=INNODB, 数据库引擎,这里是INNODB

MySQL 数据类型

MySQL数据类型介绍

  • 整形类型:
    • 一个字节的TINYINT
    • 两个字节的SMALLINT
    • 三个字节的MEDIUMINT
    • 四个字节的INT和INTEGER
    • 八个字节的BIGINT
  • 浮点数类型
    • 四个字节的FLOAT
    • 八个字节的DOUBLE
    • M+2个字节的DECIMAL(M,D): 其中M是小数点前面的, 后面的D是小数点后面的. M要 大于等于D, 又叫"定点数":在mysql中,定点数是使用字符串保存的,所以比
  • 时间类型
    • 一个字节的YEAR: 从1901年到2155年
    • 四个字节的DATE
    • 三个字节的TIME
    • 八个字节的DATETIME
    • 四个字节的TIMESTAMP, epoch的时间
  • 字符串类型
    • CHAR
    • VARCHAR (不固定的CHAR,可能实际的存储空间就是比字符串多一个字节,所以非常节省)
    • TEXT: 根据长度分成了四种:
      1. TINYTEXT
      2. TEXT
      3. MEDIUMTEXT
      4. LONGTEXT
    • ENUM类型
    • SET类型
  • 二进制类型
