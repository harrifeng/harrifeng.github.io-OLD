The Truth About Classes

What's in an Object

  • 下面是一个Object的例子 
    class MyClass
  def my_method
    @v = 1
  end
end

obj = MyClass.new
obj.class                       # MyClass

Instance Variables

  • 上面的obj就是Object,它首先包括instance varibles 
    class MyClass
  def my_method
    @v = 1
  end
end

obj = MyClass.new
# MUST CALL FIRST, otherwise, no instance_variables
obj.my_method
p obj.instance_variables        # => [:@v]
  • 和Java等静态语言不同,ruby的instance varible和object class之间没有什么联系! 比如在Java中instance variables就是定义在class里面,却在method外面的变量 
    class IronMan {
    // These are all instance variables
    public String realName;
    public String[] superPowers;
    public int age;
    // ...
}
  • 同一个类,可能不同的instance里面的instance variable的数目会不一样!!,比如下 面的代码中, 因为没有调用my_method,所以instance_variables的数目为0 
    class MyClass
  def my_method
    @v = 1
  end
end

obj = MyClass.new
p obj.instance_variables        # => []
  • 你可以把instance variable的名字和数值为hash的key和value.对每一个object来说 key和value都可以不相同.

Methods

  • object除了含有instance, 还含有method.你可以通过Object#methods()来获取obj 所有的method.大部分的class都继承了Object,所以methods()的结果都很长
  • 如果你可以看到obj的内存布局的话,你会发现obj都非常的小. 主要其只存了两个部 分(当然还有id等次要部分)
    • instance variable
    • 指向其class的reference
  • 而obj的函数则是存在class里面的,因为 
    Objects that share the same class also share the same methods, so the
methods must be stored in the class, not the object
  • 由于instance和class错综复杂的关系,所以在命名上面有很些要注意的地方:
    • obj has a "method" called my_method() 
      String.methods == "abc".methods # => false
    • MyClass has a "instance method" called my_method() 
      String.instance_methods == "abc".methods # => true
  • 正是由于object的method是其class保存的,而object的instance variable保存的, 所以不同object的instance variable不同,但是method相同

Classes Revisited

  • ruby中一个重要的概念就是: 
    Classes themselves are nothing but objects
  • 正是因为class也是一种object,所以对于object适用的特性,class也适用, class也有自己的 类型:Class 
    "hello".class                   # => String
String.class                    # => Class
  • class也肯定有method,因为class的类型是Class,那么就有 
    The methods of a class are the instance method of Class

    Class.instance_methods(false) # => [:allocate, :new, :superclass]

    String.superclass # => Object Object.superclass # => BasicObject BasicObject.superclass # => nil

    Class.superclass # => Module Module.superclass # => Object

  • 最后来看一个例子 
    class MyClass; end
obj1 = MyClass.new
obj2 = MyClass.new
  • 从上面的例子我们可以看到:obj1和obj2都是reference,这个和java中是一样的, 和 java中不一样的是:MyClass也是一个reference!只不过它是"constant"罢了(不能更改) 
    class MyClass; end
# is equals to following (with making MyClass constant)
MyClass = Class.new

Constants

  • 所有以答谢字母开头的(比如Class或者Module)reference,都是constant
  • constant有自己特殊的和文件系统紧密相连的关系:
    • 每个class或者module都是文件夹
    • 每个其他常规Constant都是其中的文件
  • 比如下面的例子 
    module MyModule
  MyConstant = 'Outer constant'

  class MyClass
    MyConstant = 'Inner constant'
  end
end
  • 它的文件系统图就是 
    MyModule
   |______MyClass
   |         |_____MyConstant
   |______MyConstant
  • 然后你就可以MyModule::MyClass::MyConstant这样来使用ruby中的constant了,正是 因为这种联系,所以ruby代码中,都是靠module和class来组织constant.就像操作系统使 用文件夹来组织文件.

  • 所以最近版本的Rake就把这些class换到了Rake的module里面

Objects and Classes Wrap-Up

  • 什么是object? 
    object只不过是一些变量的集合,另外有一个link指向class.
object的method不是生存在object里面,而是存在class里面
object的method也叫class的instance method
  • 什么是class 
    class 是一种特殊的object(an instance of Class), 所以它也有很多的变量,同时也有指向Class的link
class 的特殊性表现在它另外有许多的instance methods(为自己的instance使用), 同时还有link指向其superclass
Class的superclass是Module, 所以class也是一个module

What Happens When You Call a Method?

  • 每当你调用method的时候,ruby都要做两件事情:
    • 找到这个method,这叫做method lookup
    • 执行这个method,ruby需要一个self

Method Lookup

  • 最简单的method lookup当然就是去object里面的class link找到对应的class, 然后 method就是class的instance method之一
  • 复杂的情况就涉及到两个概念receiver和ancestor chain:
    • 所谓receiver就是调用函数(并想receive函数结果的那个物质!!), 比如my_string.reverse() 里面my_string就是receiver
    • 所谓ancestor chain就是从子类通过superclass找到父类,然后调用父类的superclass 找到父类的父类.
    • 了解这两个概念后,就知道复杂的情况下: receiver的父类没有我们要的method,然 后就要不停的ancestor chain直到找到最后的结果
  • 比如下面这个例子,寻找my_method()的过程就要ancestor chain到MyClass才能找到, 具体chain的过程可以使用ancestors来查看! 
    class MyClass
  def my_method; 'my_method()'; end
end

class MySubClass < MyClass
end

obj = MySubClass.new
obj.my_method()

MySubClass.ancestors            # => [MySubClass, MyClass, Object, Kernel, BasicObject]
  • 我们会惊奇的发现Kernel, 这个不是class而是module,也就是说虽然继承序列里面是 不包括module的,但是ancestor chain的序列里面是有module的

modules and Lookup

  • 从下面例子中我们就可以看到ruby处理module的方法:class C include了Module M, 那么ruby在解析的时候,把Module M包裹起来做成一个匿名类,然后把匿名类作为include 它的class的父类 
    module M
  def my_method
    'M#my_method()'
  end
end

class C
  include M
end

class D < C; end

D.new.my_method()               # => "M#my_method()"
D.ancestors                     # => [D, C, M, Object, Kernel, BasicObject]

The Kernel

  • 我们前面说过kernel是一个module, ruby之所以要在Object和BasicObject之间"硬生 生"的插入一个Kernel module,是为了实现这样一些函数 
    Kernel.private_instance_methods.grep(/^p/) # => [:printf, :print, :putc, :puts, :p, :proc]
  • 上面例子中的print, puts, p, 我们都很熟悉,就是"在任何地方都可以使用的函数", 为什么这些函数能够任意使用呢?看下面的例子 
    module M
  private
  def hello
    "hello in module"
  end
end

M.private_instance_methods      # => [:hello]

class C
  include M
  def hello_c
    hello
  end
end

obj = C.new
obj.hello_c                     # => "hello in module"
  • 因为定义了hello在一个"比C还大的module M"里面,所以在C里面hello就畅通无阻的 使用了. Module是一个自动include到所有代码的module(#TODO),自然能在我们所有的代 码里面直接使用!
  • 另外一个添加函数到Kernel来达到"哪里都可以直接调用这个函数"目的的做法是gem函数 
    # gems/rubygems-update-1.3.3/lib/rubygems.rb
module Kernel
  def gem(gem_name, *version_requirements)
    # ...

Method Execution

  • 前面说完了如何找到这个函数，下面该开始叙述如何执行
  • 下面有一段代码 
    def my_method
  temp = @x + 1
  my_other_method(temp)
end
  • 上面这段代码再找到函数位置后,在执行的时候,还需要解决如下两个问题:
    1. @x 这个变量是属于谁的
    2. my_other_method是谁的method
  • 这两个东西的所有者都是一个object 那就是self: 
    Every line of Ruby code is executed inside an object- the so-called current
object. you can access current object with the self keyword
  • 一旦你调用了一个函数,那么self就被设置为receiver, 知道你"明确的"使用其他object 来调用它自己的函数, 那么self就赋给那个object了
  • 下面是一个调用时候的例子 
    class MyClass
  def testing_self
    @var = 10
    my_method()
    self
  end

  def my_method
    @var = @var + 1
  end
end

obj = MyClass.new
obj.testing_self                # => #<MyClass:0x26172f0 @var=11>
  • obj开始被初始化,然后当obj调用testing_self的时候, self就成了obj, 在testing_self 内部调用my_method的时候,self还是obj. 所以就有@var的值为11.然后我们在testing_self 最后返回了self, 我们可以看到这个时候的self是一个MyClass的对象, 其instance variable @var结果为11

The Top Level

  • 前面我们讲到了,每次我们调用method的时候,self就会变成当前的object.但是当你 什么都不调用的时候,self是什么呢?在irb可以做测验 
    self                            # => main
self.class                      # => Object
  • 也就是说,一旦你开启一个Ruby程序, 你就进入了一个叫做main的object, 这个是ruby interpreter为你创建的.
  • 这个main object也叫做top-level context, 是因为你的函数返回到"最顶层"就是这 个object了
  • ruby里面的main object是一个对象,这个和Java以及C里面的main函数是没有什么关系的

Class Definitions and self

  • self在公共空间"不调用函数的时候"是被赋予了main object, 那么在某个class或者 module里面"不调用函数的时候"是被赋予了哪个object呢? 看irb 
    class MyClass
  self                          # => MyClass
  self.class                    # => Class
end
  • 可以看到,指向的是Myclass这个"object"(是Class类的object)

Module include

  • 每当在class里面include module的时候,这个module就插入到当前class的ancestor 里面,位置是class之上(紧挨着), 再插入再紧挨着class, 所以插入越早,chain里面 的位置越高 
    module Printable
end

module Document
end

class Book
  include Printable
  include Document

  ancestors                     # => [Book, Document, Printable, Object, Kernel, BasicObject]
end

Chapter 02: Tuesday: Methods

A Duplication Problem

  • 现在有一个问题要解决, 一个计算机的每个部分一旦超过99美元,就会有提示,整个系 统原来的实现是: 
    class Computer
  def initialize(computer_id, data_source)
    @id = computer_id
    @data_source = data_source
  end

  def mouse
    info = @data_source.get_mouse_info(@id)
    price = @data_source.get_mouse_price(@id)
    result = "Mouse: #{info} ($#{price})"
    return "*#{result}" if price >= 100
    result
  end

  def cpu
    info = @data_source.get_cpu_info(@id)
    price = @data_source.get_cpu_price(@id)
    result = "Cpu: #{info} ($#{price})"
    return "*#{result}" if price >= 100
    result
  end

  def keyboard
    info = @data_source.get_keyboard_info(@id)
    price = @data_source.get_keyboard_price(@id)
    result = "Keyboard: #{info} ($#{price})"
    return "*#{result}" if price >= 100
    result
  end
end

Dynamic Methods

  • 上面的例子我们可以看到,大部分函数的内容其实基本一致,这就要求我们不要重复这 么多. (代码要符合DRY原则).
  • 再ruby中做到这一点的方法是dynamic method 
    class MyClass
  def my_method(my_arg)
    my_arg * 2
  end
end

obj = MyClass.new
obj.my_method(3)                # => 6
# Same with following way
obj.send(:my_method, 3)         # => 6
  • 可能你会问使用send有什么好处. 好处是:知道最后一刻你才决定调用哪个函数.这样 可以最大限度的把函数变成"变量"一样,在循环等处使用.下面是几个使用的例子

The Camping Example

  • camping是一个框架, 它希望在runtime的时候从一配置文件中读取一些参数,然后给 这些参数赋值,比如: 
    admin : Bill
title : Rubyland
topic : Ruby and more
  • 它就使用了dynamic method, 每次读取一个配置文件的item, 然后把item的key作为 函数名, item的value作为参数 
    # Load configuration if any
if conf.rc and File.exists?(conf.rc)
  YAML.load_file(conf.rc).each do |k, v|
    conf.send("#{k}=", v)       # in detail, it will be something like conf.admin = 'Bill'
  end
end
  • 需要值得注意的是,send函数具有超强的能力,甚至可以调用private函数, 新增的 public_send是一个尊重send隐私的新send函数

The Test::Unit Example

  • unit test 框架也是一个使用send的例子, unit test框架会寻找代码中,以test 开头的函数名, 把他们存在一个数组里面,然后开始从头调用这个数组里面的函数 
    methods_names = public_instance_methods(true)
tests = methods_names.delete_if {|method_name| method_name !~ /^test./}

Defining Methods Dynamically

  • 你还可以使用defined_method来定义一个函数 
    class MyClass
  define_method :my_method do | my_arg|
    my_arg * 3
  end
end

obj = MyClass.new
obj.my_method(2)                # => 6

Step 1: Adding Dynamic Dispatches

  • 我们先使用send来简化前面的例子 
    class Computer
  def initialize(computer_id, data_source)
    @id = computer_id
    @data_source = data_source
  end

  def mouse
    component :mouse
  end

  def cpu
    component :cpu
  end

  def keyboard
    component :keyboard
  end

  def component(name)
    info = @data_source.send "get_#{name}_info", @id
    price = @data_source.send "get_#{name}_price", @id
    result = "#{name.to_s.capitalize}: #{info} ($#{price})"
    return "* #{result}" if price >= 100
    result
  end
end
  • 以mouse()为例,我们先后调用的顺序是:
    1. mouse()
    2. component()
    3. DS#get_mouse_info()
    4. DS#get_mouse_price()

Step 2: Generating Methods Dynamically

  • 上面的例子已经很简约了,但是还是有重复(几个函数), 可以使用defind_method()来进一步简化 
    class Computer
  def initialize(computer_id, data_source)
    @id = computer_id
    @data_source = data_source
  end

  def self.define_component(name)
    define_method(name) {
      info = @data_source.send "get_#{name}_info", @id
      price = @data_source.send "get_#{name}_price", @id
      result = "#{name.to_s.capitalize}: #{info} ($#{price})"
      return "*#{result}" if price >= 100
      result
    }
  end

  define_component :mouse
  define_component :cpu
  define_component :keyboard
end

Step 3: Sprinkling the Code with Introspection

  • 上面的代码已经基本没有重复了,除了最后几行. 即便是最后几行, ruby也是可以把它给去掉的! 
    class Computer
  def initialize(computer_id, data_source)
    @id = computer_id
    @data_source = data_source
    data_source.methods.grep(/^get_(.*)_info$/) { Computer.define_component $1 }
  end

  def self.define_component(name)
    define_method(name) {
      info = @data_source.send "get_#{name}_info", @id
      price = @data_source.send "get_#{name}_price", @id
      result = "#{name.to_s.capitalize}: #{info} ($#{price})"
      return "*#{result}" if price >= 100
      result
    }
  end
end
  • 这段代码较为难以理解,重点在initialize
    • 如果你把一个block传给了String#grep(), 这个block就会在"每次grep配对成功的时候"执行一次
    • grep的参数里面如果有括号,那么匹配的结果,会被存在$1里面传递给block,也就是说.
      1. data_source有get_cpu_info()这个函数,那么就会匹配cpu,然后就会给define_component 一个参数"cpu".
      2. data_source有get_mouse_info()这个函数,那么就会匹配mouse,然后就会给define_component 一个参数"mouse".
      3. 如此类推….
  • 好了,我们看到,我们没有写任何一行冗余的代码,而且更难能可贵的是 
            一旦我们引入了一个get_XXX_info函数,我们自动就拥有了XXX函数来显示其价格

    class Table
  def method_missing(id, *args, &block)
    return as($1.to_sym, *args, &block) if id.to_s =~ /^to_(.*)/
    return rows_with($1.to_sym => args[0]) if id.tos =~ /^rows_with_(.*)/
    super                       # if neither match, will go here to throw NoMethodError
  end

  # ...
end

The & Operator

  • block从某种意义上来说,是method的一个additional, anonymous参数. 大多数情况下, block只是在method里面的yield执行. 但是yield在下面两个场景里面无能为力:
    • 你想把block传递到其他的method
    • 你想把block转换成Proc
  • 解决的办法就是block 参数化'&' 
    def math(a, b)
  yield(a, b)
end

def teach_math(a, b, &operation)
  puts "Let's do the math:"
  puts math(a, b, &operation)
end

teach_math(2, 3) {|x, y| x * y }

##################################################
# <===================OUTPUT===================> #
# Let's do the math:                             #
# 6                                              #
##################################################
  • 如果想从block得到一个Proc object,那么 
    def my_method(&the_proc)
  the_proc
end

p = my_method {|name| "Hello , #{name}!"}
puts p.class
puts p.call("Bill")

# <===================OUTPUT===================> #
# Proc                                           #
# Hello , Bill!                                  #
##################################################
  • 想把Proc转换回成block的话,还是& 
    def my_method(greeting)
  puts "#{greeting}, #{yield}!"

# Hello, Bill!                                   #
##################################################
