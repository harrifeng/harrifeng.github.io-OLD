Tuesday: Methods

A Duplication Problem

  • 现在有一个问题要解决, 一个计算机的每个部分一旦超过99美元,就会有提示,整个系 统原来的实现是: 
    class Computer
  def initialize(computer_id, data_source)
    @id = computer_id
    @data_source = data_source
  end

  def mouse
    info = @data_source.get_mouse_info(@id)
    price = @data_source.get_mouse_price(@id)
    result = "Mouse: #{info} ($#{price})"
    return "*#{result}" if price >= 100
    result
  end

  def cpu
    info = @data_source.get_cpu_info(@id)
    price = @data_source.get_cpu_price(@id)
    result = "Cpu: #{info} ($#{price})"
    return "*#{result}" if price >= 100
    result
  end

  def keyboard
    info = @data_source.get_keyboard_info(@id)
    price = @data_source.get_keyboard_price(@id)
    result = "Keyboard: #{info} ($#{price})"
    return "*#{result}" if price >= 100
    result
  end
end

Dynamic Methods

  • 上面的例子我们可以看到,大部分函数的内容其实基本一致,这就要求我们不要重复这 么多. (代码要符合DRY原则).
  • 再ruby中做到这一点的方法是dynamic method 
    class MyClass
  def my_method(my_arg)
    my_arg * 2
  end
end

obj = MyClass.new
obj.my_method(3)                # => 6
# Same with following way
obj.send(:my_method, 3)         # => 6
  • 可能你会问使用send有什么好处. 好处是:知道最后一刻你才决定调用哪个函数.这样 可以最大限度的把函数变成"变量"一样,在循环等处使用.下面是几个使用的例子

The Camping Example

  • camping是一个框架, 它希望在runtime的时候从一配置文件中读取一些参数,然后给 这些参数赋值,比如: 
    admin : Bill
title : Rubyland
topic : Ruby and more
  • 它就使用了dynamic method, 每次读取一个配置文件的item, 然后把item的key作为 函数名, item的value作为参数 
    # Load configuration if any
if conf.rc and File.exists?(conf.rc)
  YAML.load_file(conf.rc).each do |k, v|
    conf.send("#{k}=", v)       # in detail, it will be something like conf.admin = 'Bill'
  end
end
  • 需要值得注意的是,send函数具有超强的能力,甚至可以调用private函数, 新增的 public_send是一个尊重send隐私的新send函数

The Test::Unit Example

  • unit test 框架也是一个使用send的例子, unit test框架会寻找代码中,以test 开头的函数名, 把他们存在一个数组里面,然后开始从头调用这个数组里面的函数 
    methods_names = public_instance_methods(true)
tests = methods_names.delete_if {|method_name| method_name !~ /^test./}

Defining Methods Dynamically

  • 你还可以使用defined_method来定义一个函数 
    class MyClass
  define_method :my_method do | my_arg|
    my_arg * 3
  end
end

obj = MyClass.new
obj.my_method(2)                # => 6

Step 1: Adding Dynamic Dispatches

  • 我们先使用send来简化前面的例子 
    class Computer
  def initialize(computer_id, data_source)
    @id = computer_id
    @data_source = data_source
  end

  def mouse
    component :mouse
  end

  def cpu
    component :cpu
  end

  def keyboard
    component :keyboard
  end

  def component(name)
    info = @data_source.send "get_#{name}_info", @id
    price = @data_source.send "get_#{name}_price", @id
    result = "#{name.to_s.capitalize}: #{info} ($#{price})"
    return "* #{result}" if price >= 100
    result
  end
end
  • 以mouse()为例,我们先后调用的顺序是:
    1. mouse()
    2. component()
    3. DS#get_mouse_info()
    4. DS#get_mouse_price()

Step 2: Generating Methods Dynamically

  • 上面的例子已经很简约了,但是还是有重复(几个函数), 可以使用defind_method()来进一步简化 
    class Computer
  def initialize(computer_id, data_source)
    @id = computer_id
    @data_source = data_source
  end

  def self.define_component(name)
    define_method(name) {
      info = @data_source.send "get_#{name}_info", @id
      price = @data_source.send "get_#{name}_price", @id
      result = "#{name.to_s.capitalize}: #{info} ($#{price})"
      return "*#{result}" if price >= 100
      result
    }
  end

  define_component :mouse
  define_component :cpu
  define_component :keyboard
end

Step 3: Sprinkling the Code with Introspection

  • 上面的代码已经基本没有重复了,除了最后几行. 即便是最后几行, ruby也是可以把它给去掉的! 
    class Computer
  def initialize(computer_id, data_source)
    @id = computer_id
    @data_source = data_source
    data_source.methods.grep(/^get_(.*)_info$/) { Computer.define_component $1 }
  end

  def self.define_component(name)
    define_method(name) {
      info = @data_source.send "get_#{name}_info", @id
      price = @data_source.send "get_#{name}_price", @id
      result = "#{name.to_s.capitalize}: #{info} ($#{price})"
      return "*#{result}" if price >= 100
      result
    }
  end
end
  • 这段代码较为难以理解,重点在initialize
    • 如果你把一个block传给了String#grep(), 这个block就会在"每次grep配对成功的时候"执行一次
    • grep的参数里面如果有括号,那么匹配的结果,会被存在$1里面传递给block,也就是说.
      1. data_source有get_cpu_info()这个函数,那么就会匹配cpu,然后就会给define_component 一个参数"cpu".
      2. data_source有get_mouse_info()这个函数,那么就会匹配mouse,然后就会给define_component 一个参数"mouse".
      3. 如此类推….
  • 好了,我们看到,我们没有写任何一行冗余的代码,而且更难能可贵的是 
            一旦我们引入了一个get_XXX_info函数,我们自动就拥有了XXX函数来显示其价格

method_missing()

  • 上面的解决方案已经比较完美,但是ruby总有多种方法做一件事情,于是就有了method_missing()
  • 先来看一个例子,如果你调用一个self不存在的函数,那么会抛出一个错误 
    class Lawyer; end

nick = Lawyer.new
nick.talk_simple

################################################################################
# <===================OUTPUT===================>                               #
# NoMethodError: undefined method `talk_simple' for #<Lawyer:0x007fa25a8812c0> #
################################################################################
  • 那这句话是编译器报的么?其实不是.是Ruby里面的method_missing()函数告诉你的.这个函数 存在于module Kernel(我们前面说过,函数调用的过程就是一个'向右移动,然后向上寻 找的过程', 而Kernel正是基本所有类的ancestor)
  • 你可以通过自己调用一下method_missing来体验一把(虽然method_missing是一个private 函数,但是我们可以使用send) 
    class Lawyer; end

nick = Lawyer.new
nick.send :method_missing, :my_method

############################################################################################
# <===================OUTPUT===================>                                           #
# NameError: undefined local variable or method `my_method' for #<Lawyer:0x007fa25a8c2d60> #
############################################################################################
  • 这里我们也可以看到method_missing就像是object的"遗书办公室",所有不知其名的函 数都会在这里被"终结"

Overriding method_missing()

  • 你可以自己重载method_missing来达到"显示重要出错信息"的作用 
    class Lawyer
  def method_missing(method, *args)
    puts "You called #{method} (#{args.join(', ')})"
    puts "(You also passed it a block)" if block_given?
  end
end

bob = Lawyer.new
bob.talk_simple('a', 'b') do
  # a block
end

##################################################
# <===================OUTPUT===================> #
# You called talk_simple (a, b)                  #
# (You also passed it a block)                   #
##################################################

Ghost Methods

  • 如果你要定义很多很多相似的函数,那么你不用一个一个的去定义,而是"不去定义", 他们找不到定义,自然就会跑到method_missing来"告状"
  • 这种让用户看起来很正常,但是在receiver端确实没有相应的method实体的函数叫做 ghost method. 下面就是一个ghost method的例子
  • 首先我们要了解一个库叫ruport,是用来做报告的 
    require 'ruport'

table = Ruport::Data::Table.new :column_names => ["country", "wine"],
:data => [["France", "Bordeaux"],
          ["Italy", "Chianti"],
          ["France", "Chablis"]]
puts table.to_text

##################################################
# <===================OUTPUT===================> #
# +--------------------+                         #
# | country |   wine   |                         #
# +--------------------+                         #
# | France  | Bordeaux |                         #
# | Italy   | Chianti  |                         #
# | France  | Chablis  |                         #
# +--------------------+                         #
##################################################
  • 在初始化完成后,我们可能要来做一些定制:比如下面,就是找到country列为"France" 那些列,然后把结果列出来. 
    require 'ruport'

table = Ruport::Data::Table.new :column_names => ["country", "wine"],
:data => [["France", "Bordeaux"],
          ["Italy", "Chianti"],
          ["France", "Chablis"]]

found = table.rows_with_country("France")
found.each do |row|
  puts row.to_csv
end

##################################################
# <===================OUTPUT===================> #
# France,Bordeaux                                #
# France,Chablis                                 #
##################################################
  • 我们看到,任务顺利的完成了,但是有个问题:ruport如何知道自己有个column叫做"country" 的呢? 不知道的话,如何生成一个叫做rows_with_country的函数
  • 真实的原因是ruport库不知道自己有一个column叫country, 而是通过ghost method的 方法来实现row_with_country和to_csv 
    class Table
  def method_missing(id, *args, &block)
    return as($1.to_sym, *args, &block) if id.to_s =~ /^to_(.*)/
    return rows_with($1.to_sym => args[0]) if id.tos =~ /^rows_with_(.*)/
    super                       # if neither match, will go here to throw NoMethodError
  end

  # ...
end
  • 从上面例子可以看出, 所谓row_with_country()就改造成了row_with(:country), 然 后to_csv就改造成了as(:csv)
