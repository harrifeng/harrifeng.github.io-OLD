Wednesday: Blocks

  • block的实质是control scope: 
    Block可以决定哪些函数和变量能够被哪些行看到
  • 有一大类的object是叫做"callable object", 它们包括:
    • blocks
    • procs
    • lambdas

How to Handle Hump Day

Back to the Basics

  • 下面是一个block的例子, "只有在调用一个method的时候,你才能创建自己的block" 
    def a_method(a, b)
  a + yield(a, b)
end

a_method(1, 2) { |x, y| (x + y) * 3} # => 10
  • block是和method同生共死的, 在method里面通过yield来执行block,并且得到返回值

The Current Block

  • 在method内部,你可以通过询问Ruby当前的call是否有个block. 这样可以防止在没有 block提供给method的情况下强行调用yield得到的错误提示 
    def a_method
  return yield if block_given?
  'no block'
end

a_method                        # => "no block"
a_method { "here's a block!" }  # => "here's a block!"

Closures

  • block是callable的object, 它需要environment(local variable, instance variable, self)等来运行.
  • 从下面的例子可以看出, block对method里面的x"直接无视", 它是从当前环境里面取x的. 这也是为什么计算机科学家把block叫做closure. closure的一大特点就是能够解决改 变"scope"的问题. 
    def my_method
  x = "Goodbye"
  yield("cruel")
end

x = "Hello"
my_method { |y| "#{x}, #{y} world" } # => "Hello, cruel world"

Scope

  • 当你使用单步调试到某个点的时候,你所能看到的local varialbe, instance variable, self等等,就是所谓的"scope"

Changing Scope

v1 = 1

class MyClass v2 = 2 local_variables # => [:v2]

def my_method v3 = 3 local_variables end

local_variables # => [:v2] end

obj = MyClass.new obj.my_method # => [:v3] obj.my_method # => [:v3] local_variables # [:v1, :obj, :_]

Scope Gates

  • Ruby只在以下几种情况下转换scope,所以,我们可以说下面几种情况都叫做"Scope Gates"
    • Class definitions
    • Module definitions
    • Methods

Flattening the Scope

  • 好了,问题来了.如果你想把数据传入到不同的scope里面怎么办? 
    my_var = "Success"

class MyClass
  # We want to print my_var here ...

  def my_method
    # ... and here
  end
end
  • 答案是使用Class.new 来替代 class, 使用define_method来代替def 
    my_var = "Success"

MyClass = Class.new do
  puts "#{my_var} in the class definition!"

  define_method :my_method do
    puts "#{my_var} in the method!"
  end
end

MyClass.new.my_method

##################################################
# <===================OUTPUT===================> #
# Success in the class definition!               #
# Success in the method!                         #
##################################################

instance_eval()

  • 函数instance_eval非常强大,可以看到某个instance的private的内容, 不仅仅可以看, 而且海可以改 
    class MyClass
  def initialize
    @v = 1
  end
end

obj = MyClass.new
obj.instance_eval do
  p self                        # => #<MyClass:0x007fd052173778 @v=1>
  p @v                          # => 1
end

v = 2
obj.instance_eval { @v = v }
obj.instance_eval { @v }        # => 2

Callable Objects

Proc Objects

  • 为了让"一切都是object", Ruby为block设计了Proc class. 你可以如下设计一个block object 
    inc = Proc.new { |x| x + 1 }
inc.call(2)                     # => 3

The & Operator

  • block从某种意义上来说,是method的一个additional, anonymous参数. 大多数情况下, block只是在method里面的yield执行. 但是yield在下面两个场景里面无能为力:
    • 你想把block传递到其他的method
    • 你想把block转换成Proc
  • 解决的办法就是block 参数化'&' 
    def math(a, b)
  yield(a, b)
end

def teach_math(a, b, &operation)
  puts "Let's do the math:"
  puts math(a, b, &operation)
end

teach_math(2, 3) {|x, y| x * y }

##################################################
# <===================OUTPUT===================> #
# Let's do the math:                             #
# 6                                              #
##################################################
  • 如果想从block得到一个Proc object,那么 
    def my_method(&the_proc)
  the_proc
end

p = my_method {|name| "Hello , #{name}!"}
puts p.class
puts p.call("Bill")

##################################################
# <===================OUTPUT===================> #
# Proc                                           #
# Hello , Bill!                                  #
##################################################
  • 想把Proc转换回成block的话,还是& 
    def my_method(greeting)
  puts "#{greeting}, #{yield}!"
end

my_proc = proc {"Bill"}
my_method("Hello", &my_proc)

##################################################
# <===================OUTPUT===================> #
# Hello, Bill!                                   #
##################################################
