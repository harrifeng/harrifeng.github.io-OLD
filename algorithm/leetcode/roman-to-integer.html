题目

  • 输入一个罗马数字表示法(1到3999),返回它的正整数表示方法 
    Sample Input: MCMLXXX
Sample Output: 1980

思路

  • 罗马数字是一个以五为小单位的计数:
    I V X L C D M
    1 5 10 50 100 500 1000
  • 和integer to roman 一样,存在难点:数字的位置不同代表不同的结果,比如IV和VI的值是不一样的. 所以要分成一下四个段进行考虑:
    • 小于4
    • 等于4
    • 大于4并且小于8
    • 等于9
  • 在处理IV和VI不同的理解方面, Roman to Integer可以更加的从容, 因为我可以通过比较两个char来判定怎么办,比如:
    1. 当前字符和前面一个字符为II :相等,那么就简单相加
    2. 当前字符和前面一个字符为VI :前面的大,那么还是简单相加
    3. 当前字符和前面一个字符为IV :前面的小,那么就是相减, 而且要减两倍,因为处理到V的时候,前面的I的值已经加过一次了
    4. 当前为第一个字符:简单相加
  • 通过分析我们发现,上面四种情况,可以只分成两种: 1当前字符比前面的小(减去两倍的小的值)和2其他情况(简单相加

解答

public class Solution {
    private int cton(char c) {
        switch(c) {
            case 'I' : return 1;
            case 'V' : return 5;
            case 'X' : return 10;
            case 'L' : return 50;
            case 'C' : return 100;
            case 'D' : return 500;
            case 'M' : return 1000;
            default: return 0;
        }
    }
    public int romanToInt(String s) {
        int ret = 0;
        for (int i = 0; i < s.length(); i++) {
            if (i > 0 && (cton(s.charAt(i)) > cton(s.charAt(i-1)))) {
                ret += cton(s.charAt(i))  - 2 * cton(s.charAt(i-1));
            } else {
                ret += cton(s.charAt(i));
            }
        }
        return ret;
    }
}
